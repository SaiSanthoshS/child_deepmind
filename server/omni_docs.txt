<p>Gemini 3.5 Flash is <strong>generally available (GA)</strong>, stable, and ready for scaled
production use. As our most intelligent Flash model, it delivers sustained
frontier performance in agentic execution, coding, and long-horizon tasks at
scale.</p>

<p>This guide contains an overview of improvements, API changes, and migration
guidance for Gemini 3.5 Flash.</p>

<h2 id="whats-new" data-text="What's new" tabindex="-1">What's new</h2>

<ul>
<li><strong>Sustained frontier performance:</strong> Our most intelligent Flash model,
optimized for agentic and coding tasks at scale.</li>
<li><strong>Agentic execution:</strong> Sub-agent deployment, problem solving, and rapid
agentic loops at scale.</li>
<li><strong>Coding:</strong> Iterative coding cycles, rapid exploration, and prototyping to
test alternate paths and dynamically explore solutions.</li>
<li><strong>Long horizon:</strong> Multi-step workflows and tool use at scale.</li>
<li><strong>Thought preservation:</strong> The model maintains intermediate reasoning across
multi-turn conversations automatically. No API changes needed.</li>
<li><strong>New default effort level:</strong> Default thinking effort changed from <code translate="no" dir="ltr">high</code> to
<code translate="no" dir="ltr">medium</code>. See <a href="#default-effort-level">New default effort level</a> for details.</li>
<li><strong>Improved <code translate="no" dir="ltr">low</code> thinking:</strong> <code translate="no" dir="ltr">low</code> is now significantly improved for code
and agentic tasks that require fewer steps, offering strong quality at lower
latency and cost.</li>
<li><strong>GA release:</strong> Stable model for scaled production use.</li>
</ul>

<h3 id="choose-model" data-text="Choosing the right Flash model" tabindex="-1">Choosing the right Flash model</h3>

<p>Gemini 3.5 Flash is our most intelligent and capable Flash model. However,
different use cases may have different cost and latency requirements.</p>

<ul>
<li><strong>Gemini 3.1 Flash-Lite</strong>: For low cost,
high-volume tasks that don&#39;t require the advanced reasoning depth of 3.5
Flash, we recommend using <a href="/gemini-api/docs/models/gemini-3.1-flash-lite">Gemini 3.1 Flash-Lite</a>.
It is a stable, long-term model optimized for efficiency. See the
<a href="/gemini-api/docs/models/gemini-3.1-flash-lite#developer_guide">Flash-Lite developer guide</a>
for more details.</li>
<li><strong>Gemini 3 Flash Preview</strong>: While we recommend migrating to 3.5 Flash for
GA stability and improved reasoning, <a href="/gemini-api/docs/models/gemini-3-flash-preview">Gemini 3 Flash (Preview)</a>
remains available for developers who want to continue testing with the
preview model.</li>
</ul>

<h2 id="behavioral-changes" data-text="Behavioral changes" tabindex="-1">Behavioral changes</h2>

<h3 id="default-effort-level" data-text="New default effort level: medium" tabindex="-1">New default effort level: <code translate="no" dir="ltr">medium</code></h3>

<p>The default thinking effort is now <strong><code translate="no" dir="ltr">medium</code></strong>, changed from <code translate="no" dir="ltr">high</code> in Gemini 3
Flash Preview. <code translate="no" dir="ltr">medium</code> yields very good results across a wide range of tasks
while being faster and more cost-efficient. For complex problems, <code translate="no" dir="ltr">high</code>
encourages the model to think more deeply.</p>

<table>
<thead>
<tr>
<th style="text-align: left">Effort level</th>
<th style="text-align: left">When to use</th>
</tr>
</thead>

<tbody>
<tr>
<td style="text-align: left"><code translate="no" dir="ltr">minimal</code></td>
<td style="text-align: left">Optimized for response speed. Chat-like use cases, quick factual answers, simpler tool calls.</td>
</tr>
<tr>
<td style="text-align: left"><code translate="no" dir="ltr">low</code></td>
<td style="text-align: left">Code and agentic tasks that require lower latency and fewer steps. Also works well for analysis and writing tasks that require some thinking.</td>
</tr>
<tr>
<td style="text-align: left"><strong><code translate="no" dir="ltr">medium</code> (default)</strong></td>
<td style="text-align: left"><strong>Best quality for most tasks. Recommended for complex code and agentic use cases.</strong></td>
</tr>
<tr>
<td style="text-align: left"><code translate="no" dir="ltr">high</code></td>
<td style="text-align: left">Maximizes the model&#39;s ability to think and use tools. Best for complex reasoning, hard math, and the most difficult code or agent tasks. Allows extended thoughts and function calls.</td>
</tr>
</tbody>
</table>

<h3 id="thought-preservation" data-text="Thought preservation" tabindex="-1">Thought preservation</h3>

<p>The model maintains intermediate reasoning across multi-turn conversations
automatically. When present in the conversation history, reasoning context
carries forward, which improves performance on complex multi-step tasks
like iterative debugging and code refactoring. No API changes needed:</p>

<ul>
<li><strong>Interactions API</strong>: Thoughts are already preserved automatically. No
change in behavior.</li>
<li><strong>GenerateContent API</strong>: Beginning with Gemini 3.5 Flash, the model uses
reasoning context from all previous turns when thought signatures are
present in the conversation history. To enable this, pass the full,
unmodified conversation history (including
<a href="/gemini-api/docs/thought-signatures">thought signatures</a>) in <code translate="no" dir="ltr">contents</code>.
The SDKs handle this automatically.</li>
</ul>

<h2 id="parameter-updates" data-text="Parameter updates and best practices in Gemini 3.x" tabindex="-1">Parameter updates and best practices in Gemini 3.x</h2>

<p>The following apply to <strong>all Gemini 3.x models</strong>, including Gemini 3.5 Flash.</p>

<ul>
<li><strong><code translate="no" dir="ltr">temperature</code>, <code translate="no" dir="ltr">top_p</code>, <code translate="no" dir="ltr">top_k</code></strong>: we strongly recommend not changing the
default values. Gemini 3&#39;s reasoning capabilities are optimized for the
default settings.</li>
<li>Use <code translate="no" dir="ltr">thinking_level</code> instead of <code translate="no" dir="ltr">thinking_budget</code>.</li>
<li><strong>Function calling response matching</strong>: <code translate="no" dir="ltr">id</code>, <code translate="no" dir="ltr">name</code>, and response count
must match the preceding calls.</li>
<li><strong>Multimodal function responses</strong>: include multimodal content inside the
function response, not outside it.</li>
<li><strong>Inline instructions in function responses</strong>: append to the function
response text, not as separate parts.</li>
<li><strong>Reduce unnecessary tool calls</strong>: Use lower thinking levels or experiment
with system instructions to reduce tool calls in agentic workflows.</li>
</ul>

<p>See the sections below for how to update your code.</p>

<h3 id="reducing-tool-calls" data-text="Reducing unnecessary tool calls" tabindex="-1">Reducing unnecessary tool calls</h3>

<p>If you experience an overuse of tool calls, two techniques help minimize them:</p>

<ol>
<li><p><strong>Start by reducing the thinking level</strong> (<code translate="no" dir="ltr">medium</code>, <code translate="no" dir="ltr">low</code>, or <code translate="no" dir="ltr">minimal</code>):
Higher thinking levels encourage the model to use more tools to explore and
verify, so lowering the level can reduce tool calls.</p></li>
<li><p><strong>Add a system instruction:</strong> If overuse persists after adjusting the
thinking level, consider a prompt that restricts tool usage. For example:</p>
<div></div><devsite-code><pre class="devsite-click-to-copy" translate="no" dir="ltr" is-upgraded syntax="Text only"><code translate="no" dir="ltr">You have a limited action budget of &lt;n&gt; tool calls. Use them efficiently.
</code></pre></devsite-code></li>
</ol>

<h2 id="migration" data-text="Migration checklist" tabindex="-1">Migration checklist</h2>
<aside class="note"><strong>Note:</strong><span> <strong>Automate this migration with a coding agent.</strong> If you use a coding
agent that supports skills (like Antigravity), install the
<a href="/gemini-api/docs/coding-agents#gemini-interactions-api">Gemini Interactions API skill</a> and run:
<code translate="no" dir="ltr">/gemini-interactions-api migrate my app to Gemini 3.5 Flash</code></span></aside>
<p><strong>We strongly recommend updating to <code translate="no" dir="ltr">google-genai</code> SDK v2.0.0 or later.</strong> This
version introduces breaking changes to the Interactions API. See the
<a href="/gemini-api/docs/interactions-breaking-changes-may-2026">breaking changes migration guide</a> for details.</p>

<h3 id="migrate-from-3-flash-preview" data-text="Migrate from Gemini 3 Flash Preview" tabindex="-1">Migrate from Gemini 3 Flash Preview</h3>

<ul>
<li>Update model name: <code translate="no" dir="ltr">gemini-3-flash-preview</code> → <code translate="no" dir="ltr">gemini-3.5-flash</code></li>
<li>Review pricing. Gemini 3.5 Flash is more expensive than Gemini 3 Flash
Preview. If your use case is highly cost-sensitive, consider migrating to
<a href="/gemini-api/docs/models/gemini-3.1-flash-lite">Gemini 3.1 Flash-Lite</a>
instead. See the <a href="/gemini-api/docs/pricing#gemini-3.5-flash">pricing page</a> for details.</li>
<li>Remove <code translate="no" dir="ltr">temperature</code>, <code translate="no" dir="ltr">top_p</code>, <code translate="no" dir="ltr">top_k</code> from your config (no longer
recommended).</li>
<li>Replace <code translate="no" dir="ltr">thinking_budget</code> with <code translate="no" dir="ltr">thinking_level</code>.</li>
<li>Add <code translate="no" dir="ltr">id</code> and matching <code translate="no" dir="ltr">name</code> to all <code translate="no" dir="ltr">FunctionResponse</code> parts.</li>
<li>Test your prompts. Default effort changed from <code translate="no" dir="ltr">high</code> → <code translate="no" dir="ltr">medium</code>; verify
quality, speed, and cost.</li>
<li>Thought preservation is now on by default. Reasoning context carries forward
across turns, which improves performance but may increase token usage.</li>
<li>Reduce unnecessary tool calls: start by reducing the thinking level
(<code translate="no" dir="ltr">medium</code>, <code translate="no" dir="ltr">low</code>, or <code translate="no" dir="ltr">minimal</code>); add a system instruction to constrain tool
usage if overuse persists.</li>
<li><a href="/gemini-api/docs/computer-use">Computer Use</a> is supported.</li>
</ul>

<h3 id="migrate-from-25" data-text="Migrate from Gemini 2.5" tabindex="-1">Migrate from Gemini 2.5</h3>

<p>All of the above, plus:</p>

<ul>
<li>Simplify prompts. If you used chain-of-thought prompt engineering to force
reasoning, try <code translate="no" dir="ltr">thinking_level: &quot;medium&quot;</code> or <code translate="no" dir="ltr">&quot;high&quot;</code> with simpler prompts
instead.</li>
<li>Test PDF and media workloads. If you relied on specific behavior for dense
document parsing, test the <code translate="no" dir="ltr">media_resolution_high</code> setting to ensure
continued accuracy. Migrating to Gemini 3 defaults may also increase token
usage for PDFs but decrease it for video; if requests exceed the context
window, explicitly reduce the <code translate="no" dir="ltr">media_resolution</code>. See the
<a href="/gemini-api/docs/media-resolution">media resolution</a> docs for
details.</li>
<li>Leverage <a href="/gemini-api/docs/tool-combination">combined tool use</a>.
Google Search, URL context, code execution, and custom functions can be used
in the same request.</li>
<li>If using multimodal function responses, move multimodal content inside
function response parts, not alongside them.</li>
<li>If using inline instructions with function responses, append them to the
function response text separated by two newlines, not as separate parts.</li>
<li>Image segmentation is not supported in Gemini 3.x. For segmentation
workloads, continue using Gemini 2.5 Flash with thinking off, or
<a href="/gemini-api/docs/robotics-overview">Gemini Robotics-ER 1.6</a>.</li>
<li>Remove <code translate="no" dir="ltr">candidate_count</code> from your config (not supported in Gemini 3.x)</li>
</ul>

<h2 id="gemini-3-features" data-text="Gemini 3 family features" tabindex="-1">Gemini 3 family features</h2>

<p>Gemini 3.5 Flash inherits all Gemini 3 family capabilities, including Computer Use.
Features introduced in Gemini 3 that carry forward:</p>

<ul>
<li><a href="/gemini-api/docs/thinking"><strong>Thinking</strong></a><strong>:</strong> Encrypted
reasoning context preserved across API calls. Automatic in the Interactions
API; implicit in GenerateContent.</li>
<li><a href="/gemini-api/docs/structured-output"><strong>Structured outputs with tools</strong></a><strong>:</strong> Combine JSON mode with
built-in tools (Search, URL context, code execution, function calling).</li>
<li><a href="/gemini-api/docs/function-calling#multimodal"><strong>Multimodal function responses</strong></a><strong>:</strong> Return
images, audio, and other media in function call results.</li>
<li><a href="/gemini-api/docs/code-execution#images"><strong>Code execution with images</strong></a><strong>:</strong> Execute code that
processes and generates images.</li>
<li><a href="/gemini-api/docs/tool-combination"><strong>Combined tool use</strong></a><strong>:</strong> Use built-in tools and
custom function calling in the same request.</li>
<li><a href="/gemini-api/docs/media-resolution"><strong>Media resolution</strong></a><strong>:</strong>
Fine-grained control over token allocation for image, video, and PDF inputs.
Gemini 3 models support per-content-item resolution settings (<code translate="no" dir="ltr">low</code>,
<code translate="no" dir="ltr">medium</code>, <code translate="no" dir="ltr">high</code>, <code translate="no" dir="ltr">ultra_high</code>) for mixed-fidelity prompts.</li>
<li><a href="/gemini-api/docs/thought-signatures"><strong>Thought signatures</strong></a><strong>:</strong>
Encrypted representations of the model&#39;s internal reasoning. Required for
multi-turn function calling in stateless mode; managed automatically by the
Interactions API and the official SDKs.</li>
</ul>

<h2 id="prompting-best-practices" data-text="Prompting best practices" tabindex="-1">Prompting best practices</h2>

<p>Gemini 3.x models are reasoning models, which changes how you should prompt.</p>

<ul>
<li><strong>Precise instructions:</strong> Be concise. Gemini 3.x responds best to direct,
clear instructions. Verbose or complex prompt engineering techniques designed
for older models may cause the model to over-analyze.</li>
<li><strong>Output verbosity:</strong> By default, Gemini 3 and 3.1 is less verbose and prefers
direct, efficient answers. If your use case requires a conversational tone,
steer the model explicitly in your prompt (for example, &quot;Explain this as a
friendly, talkative assistant&quot;).</li>
<li><strong>Context management:</strong> When working with large datasets (such as entire
books, codebases, or long videos), place your specific instructions or
questions at the end of the prompt, after the data context. Anchor the
model&#39;s reasoning by starting your question with a phrase like, &quot;Based on
the preceding information...&quot;.</li>
</ul>

<p>Learn more about prompt design strategies in the
<a href="/gemini-api/docs/prompting-strategies">prompt engineering guide</a>.</p>

<h2 id="limitations" data-text="Limitations" tabindex="-1">Limitations</h2>

<ul>
<li>Image segmentation is not supported in Gemini 3.x. For segmentation
workloads, continue using Gemini 2.5 Flash with thinking off, or
<a href="/gemini-api/docs/robotics-overview">Gemini Robotics-ER 1.6</a>.</li>
</ul>

<h2 id="faq" data-text="FAQ" tabindex="-1">FAQ</h2>

<ol>
<li><p><strong>What is the knowledge cutoff for Gemini 3.5 Flash?</strong> Gemini 3.5 Flash
has a knowledge cutoff of January 2025. For more recent information, use the
<a href="/gemini-api/docs/google-search">Search Grounding</a> tool.</p></li>
<li><p><strong>What are the context window limits?</strong> Gemini 3.5 Flash supports a 1
million token input context window and up to 65k output tokens.</p></li>
<li><p><strong>Will my old <code translate="no" dir="ltr">thinking_budget</code> code still work?</strong> Yes, <code translate="no" dir="ltr">thinking_budget</code>
is still supported for backward compatibility, but we recommend migrating to
<code translate="no" dir="ltr">thinking_level</code> for more predictable performance. Don&#39;t use both in the
same request.</p></li>
<li><p><strong>Does Gemini 3.5 Flash support the Batch API?</strong> Yes. See the
<a href="/gemini-api/docs/batch-api">Batch API</a> guide for details.</p></li>
<li><p><strong>Is Context Caching supported?</strong> Yes,
<a href="/gemini-api/docs/caching">Context Caching</a> is supported.</p></li>
<li><p><strong>Which tools are supported?</strong> Gemini 3.5 Flash supports
<a href="/gemini-api/docs/google-search">Google Search</a>,
<a href="/gemini-api/docs/maps-grounding">Grounding with Google Maps</a>,
<a href="/gemini-api/docs/file-search">File Search</a>,
<a href="/gemini-api/docs/code-execution">Code Execution</a>,
<a href="/gemini-api/docs/url-context">URL Context</a>, and
standard <a href="/gemini-api/docs/function-calling">Function Calling</a>,
including
<a href="/gemini-api/docs/tool-combination">combined tool use</a>.
<a href="/gemini-api/docs/computer-use">Computer Use</a> is <strong>not</strong>
supported in Gemini 3.5 Flash.</p></li>
</ol>
